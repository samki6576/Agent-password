HOW IT WORKS
================================================================================

1. USER LOGS IN
   - Clicks "Connect Google Calendar"
   - Redirected to Auth0 OAuth
   - Auth0 returns authorization code
   - Code exchanged for access_token
   - Token stored in HTTP-only cookie

2. AGENT CALLS GATEWAY
   - Agent: GET /api/get-events (no token in request)
   - Request includes cookies automatically
   - Gateway retrieves token from secure cookie
   - Gateway checks token expiry
   - Gateway refreshes if expired

3. GATEWAY CALLS GOOGLE
   - Uses token to call Google Calendar API
   - Returns events to agent
   - Token never exposed to agent

4. COMPLETE FLOW
   User → Auth0 → HTTP-Only Cookie → Gateway → Google → Agent
   
   Agent never sees the token. Token stays secure. Done.
